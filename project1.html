Navigation on Scooter
Redesign a social dashboard to a navigation dashboard for scooters.
Introduction
Navigation on Scooter
This is a device dashboard for the scooter. It was developed as a feature with a social purpose. Users can design their creations and sync them to the scooter dashboard. At the same time, they can also share their designs on social platforms.
The navigation function is an upgraded version of this dashboard. Therefore, there are a lot of restrictions and limitations to overcome.
​
My mission was to create a good navigation experience for the user. This dashboard shows turn-by-turn instructions while routing when the phone and scooter are connected via Bluetooth. The challenge is making information simple and letting riders access it safely. I managed the entire UX and UI design from beginning to end.
Mission
Add the navigation feature to the existing scooter dashboard.
The goal is to let the rider understand the next direction with a glance in one second.
Challenge
The navigation function is a crucial selling point for the vehicle. Unfortunately, it can not change the hardware due to the alpha version already being sold. I need to re-design this structure via software changes and build a navigation function into it.
My Role
I am responsible for the entire UX and UI design, from defining the problem to delivering the final product to go to market.
​
Mar 2018 - Sep 2019
The Problem
To Make Route Instruction In A Glance For Riders
Background
On the hardware side, there are only three buttons for interaction, and the bandwidth of Bluetooth is limited. The file size should be small while transferring. From the marketing perspective, adding hardware in the new update is impossible.
On the firmware side, It has the original interaction structure in the first version. Considering the user experience will be harmful with a massive update. The navigation function should follow the original architecture.
On the app side, the app's information architecture is not proper for navigation usage. Therefore, we should build up another structure to put map purchase, route settings, and navigation.
Finding
Through discovering and brainstorming with the members, we got some possible directions to design the navigation. The limitations: it is a simple device, so the computing of the dashboard is not robust. It is impossible to show the navigation view that google Maps or portable car GPS navigation systems have.
The QUESTION
How to Make Riders Know the Right Corner to Turn
We got some ideas after several discussions. The ideas are as follows:
1
Streetview Indicator: display the Streetview on-screen before 10 miles of the corner.
2
Ring Indicator: display a ring as a countdown timer before the turn.
3
Number Indicator: display the distance before the junction.
4
Background Indicator 1: decrease the background color for the turning of time.
5
Background Indicator 2: notify the user by changing the background color.
Testing
We were not sure which one is best for riders, therefore we made rapid prototypes to test if testers could reach the defined destination without verbal instructions. Results are as follows:
Streetview Indicator
The screen displays the Street View 10 meters before the corner. The rider cannot carefully read the details on the map. It is very dangerous for the rider and difficult to notify them to take the next action.
Ring Indicator
It displays a ring as a countdown timer before the turn. This is an abstract concept for riders. It is important to provide a clear indication for the rider while driving. Anything that requires them to think twice introduces danger. Therefore, the ring countdown is not in the selection.
​Background Indicator 2
It notifies the user by changing the background color before the turning corner. The rider said: the didn’t see it and missed the corner.
Number Indicator
It displays the distance before the junction. The rider mentioned it is very clear for them to know when to take action, even though it is not the most beautiful visual design among all the prototypes.
​Background Indicator 1
It decreases the background color for the turning of time. The rider said: the didn’t notify it and missed the corner.
The Solution
The rider cannot be distracted when they are driving. Therefore, the Number Indicator provides the clearest and best instruction for riders.
The rider can't get the message we want them to know with subtle indicators. Therefore, the best option is the most specific one—the Number Indicator. It is the clearest for the rider so they can understand the next action in a glance.
the design
Wireframe & UI Flow
Using the feedback gathered from testing and discussions with teams, we worked together to create wireframes with proposed solutions for the critical issues. After alignment with teams, we converted the wireframes to UI flows.
Design System
Considering the consistency with scooter brand and the previous version. We decided to keep the app color in the red. The blue and dark gray are the secondary and helping colors.
Final Design
To work with the firmware and software engineering teams, we ran several important iterations to make sure the design was developable.
