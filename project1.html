Navigation on Scooter
Redesign a social dashboard to a navigation dashboard for scooters.
Introduction
Navigation on Scooter
It is a vice dashboard for the scooter. It developed as a feature with a social purpose. Users can design their creations and sync them into the scooter dashboard. At the same time, they can also share their designs on social platforms.
The navigation function is the upgraded version for this dashboard. Therefore, there are a lot of restrictions and limitations to go over.
​
My mission was to create a good navigation experience for the user. This dashboard will show turn-by-turn instructions while routing when the Bluetooth-connected phone and scooter. The challenge is making information simple and letting riders reach it in safe. I manage the entire UX and UI design from beginning to end.
Mission
Add the navigation feature to the existed scooter dashboard.
The goal is to let the rider get to the next direction with a glance in one second.
Challenge
The navigation function is a crucial selling point for the vehicle. Unfortunately, it can not change the hardware due to the alpha version already being sold. I need to re-design this structure via software changes and build a navigation function into it.
My Role
I am responsible for the entire UX and UI design, from defining the problem to delivering the final product to go to market.
​
Mar 2018 - Sep 2019
The Problem
To Make Route Instruction In A Glance For Riders
Background
On the hardware side, there are only three buttons for interaction, and the bandwidth of Bluetooth is limited. The file size should be small while transferring. From the marketing perspective, adding hardware in the new update is impossible.
On the firmware side, It has the original interaction structure in the first version. Considering the user experience will be harmful with a massive update. The navigation function should follow the original architecture.
On the app side, the app's information architecture is not proper for navigation usage. Therefore, we should build up another structure to put map purchase, route settings, and navigation.
Finding
Through discovering and brainstorming with the members, we got some possible directions to design the navigation. The limitations: it is a simple device, so the computing of the dashboard is not robust. It is impossible to show the navigation view that google Maps or portable car GPS navigation systems have.
The QUESTION
How to Make rider know the right corner to turn
We got some ideas after several discussions. The ideas in the following:
1
Streetview Indicator: display the Streetview on-screen before 10 miles of the corner.
2
Ring Indicator: display a ring as a count-down timer before the turn.
3
Number Indicator: display the distance before the junction.
4
Background Indicator 1: decrease the background color for the turning of time.
5
Background Indicator 2: notify the user by changing the background color.
Testing
We were not sure which one is best for riders, therefore we made rapid prototypes to test if the tester can go to the defined destination thought out by not telling a word. Result as the following:
Streetview Indicator
The screen displays the Streetview before 10 miles of the corner. The rider can not carefully read the detail on the map. It is very dangerous for the rider and hard to notify him to do the next action
Ring Indicator
It displays a ring as a count-down timer before the turn. It is an abstract concept for riders. It is important to make a specific idea for the rider during driving. Anything that needs them to think twice introduces danger. So that ring count down is not in the selection.
​Background Indicator 2
It notifies the user by changing the background color before the turning corner. The rider said: the didn’t see it and missed the corner.
Number Indicator
display the distance before the junction. The rider mentioned it is very clear for them to know when to take action. Even it is not a beautiful visual design in all the sets of prototypes.
​Background Indicator 1
It decreases the background color for the turning of time. The rider said: the didn’t notify it and missed the corner.
The Solution
The rider can not be distracted when they are driving, Therefore the number Indicator has made a the clearest and best instruction for rider
The rider can't get the message we went them to know. so that the best is the most specific one — The number Indicator; it is the clearest for the rider, that they can get the following action idea in a glance.

Design System

Final Design
