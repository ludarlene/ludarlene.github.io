Creating an Intuitive Motorcycle Dashboard
Collaborative Design Process
Introduction
Motor Dashboard
It is the main dashboard for high-level motorcycles. The challenge is to build an electric speedometer with an easy-using and solves the limited traffic regulations with beautiful appearance. I managed the UX part early and completed the production design.
Mission
Add the navigation feature to the existing scooter dashboard.
The goal is to let the rider get to the next direction with glance in one second.
Challenge
For the traffic regulations, some information should be shown on the dashboard continuously during riding. Placing this info and managing dynamic dat(ex., following route ) in small display takes work. As well as the app should be re-designed to fill multiple dashboards management.
My Role
Information architect, UX design, Implementation

Oct 2019 - Sep 2021

I worked closely with one Program Manager, one graphic designer, one firmware engineer, two software engineers, and 1 Qengineer.
The Problem
The Dashboard UX Consistency and Traffic Regulation
Background
We already have dashboard product which is 5” square display, and the following product is 7” inch ratio 4:3 display. It should use the same app to control and show the navigation. Dislike the 5” display is vice dashboard, this 7” display is central dashboard embed in heavy motorcycle.

The dashboard UX consistency should be maintained by making sure that the same app is used to control and show the navigation on both the 5” and 7” displays.
The Traffic Regulation for Dash Board
Some information MUST be shown on the screen all the time during riding. Therefore, I got list of the icons for traffic regulations, ensuring the rider can see these icons at any time during any riding moment.
The new design should fit into the original app and dashboard system. The app's user experience should be the same between the original small displays and the new 5” display.
Rapid prototype
Rapid Prototyping and User Feedback
I designated specific areto display the navigation information, while the remaining space was allocated for the traffic information, which would replace the traditional LED clusters. We used wireframes to communicate with stakeholders and ensure that all the elements were in the right place and intuitive to use.

In contrast to the original design, the target motorcycle featured additional interaction hardware, specifically handle with more buttons. I arranged these buttons to enable more interaction and functionality. Therefore, it was necessary to create rapid prototype to simulate the interactive behavior.

We tested this prototype with five internal users and received feedback that helped us refine the user interface functions.
the design
Technology Knights Style
To differentiate between fuel vehicles and electric vehicles, we used different icons on the dashboard. We took the time to carefully confirm and design the appropriate elements. During this process, we tested prototypes, had discussions with teams, and collaborated to create wireframes and propose solutions for critical issues. Finally, we confirmed the wireframe and UI flow.
For the visual design, we adopted futuristic style inspired by technology knights and incorporated the original instrument design to enhance the technological feel of the heavy motorcycle.
